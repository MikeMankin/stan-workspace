# Why Daily Standups (Ed Edition)

## The Problem
Right now, visibility into Ed's work comes from:
- Hunting through GitHub PRs
- Asking "what's done?"
- Waiting for blocked work to surprise us
- No clear record of daily progress

This burns time and creates surprises.

## The Solution
One daily markdown file. 5-10 minutes. Done.

**What it fixes:**
- **Stan** reads during morning heartbeat → knows what's moving + what's stuck
- **Miguel** scans for blockers → unblocks immediately
- **Professor** gets context on velocity + architecture decisions
- **Team** has a running log of why decisions were made

## Format Philosophy
- Keep it lean (not a novel)
- Useful for *you* first (not for us to judge)
- Real blockers get real priority (not buried in noise)
- Decisions get context (so future-you doesn't forget why)

## Where It Lives
`~/coding/progress/YYYY-MM-DD.md`

One file per day. Simple chronological log.

## What Success Looks Like
- Stan can read it in 2 minutes
- Miguel can spot blockers in 30 seconds
- Professor can track velocity trends
- You can look back and see *why* you built things a certain way

## No Judgment
This is not a performance metric. It's a communication tool.
If you shipped 10 lines or 10,000 — the file matters either way.
If you're blocked on something, say it. That's the whole point.
